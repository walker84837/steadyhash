# Contributing to steadyhash

First off, thank you for considering contributing to steadyhash! It's people like you that make open source such a great community.

## Code of Conduct

This project and everyone participating in it is governed by the [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How can I contribute?

### Reporting bugs

You can report a bug by [opening a new issue](https://github.com/walker84837/steadyhash-rs/issues/new?assignees=&labels=bug&projects=&template=bug_report.md&title=). Please use the "Bug report" template and provide as much detail as possible.

### Suggesting enhancements

If you have an idea for a new feature or an improvement to an existing one, you can suggest it by [opening a new issue](https://github.com/walker84837/steadyhash-rs/issues/new?assignees=&labels=enhancement&projects=&template=feature_request.md&title=). Please use the "Feature request" template.

### Pull Requests (PRs)

We love pull requests! To submit one, please follow these steps:

1.  **Fork the repository** and create a feature branch.
2.  **Make your changes**.
3.  **Ensure the test suite passes**: Run `cargo test` to make sure your changes haven't broken anything.
4.  **Format your code**: Run `cargo fmt` to ensure your code adheres to the project's style.
5.  **Lint your code**: Run `cargo clippy` to check for common mistakes and style issues.
6.  **Submit a pull request**: Your PR should be clear and concise, and it should reference any issues it fixes. Please use the pull request template provided.

We will review your pull request as soon as possible. Thank you for your contribution!
